## Events

Use the `@press` event handler to handle interactions via mouse, keyboard, and touch. This is similar to `@click`, but normalized for consistency across browsers, devices, and interaction methods. Read our [blog post](blog/building-a-button-part-1) to learn more.

The `@press-start`, `@press`, and `@press-end` events each receive a <TypeLink links={typesDocs.links} type={typesDocs.exports.PressEvent} />, which provides information about the target and interaction method. The `@press-change` event receives the current pressed state as a boolean. See [usePress](usePress) for more details.

## Pending

Use the `isPending` prop to display a pending state. Pending buttons remain focusable, but are otherwise disabled. `VueButton` renders a built-in spinner with `role="progressbar"` while pending so the state stays exposed to assistive technologies.

## Link buttons

The `VueButton` component always represents a button semantically. To create a link that visually looks like a button, use the `VueLink` component instead. You can reuse the same styles you apply to the `VueButton` component on `VueLink`.
